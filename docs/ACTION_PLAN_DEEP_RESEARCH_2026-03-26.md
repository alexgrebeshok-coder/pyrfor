## 8. Реестр отечественного ПО

### Почему это стратегически важно

1. **Госзакупки:** Без реестра ты не можешь участвовать в тендерах (44-ФЗ, 223-ФЗ). А это — Росавтодор, ФАД, региональные министерства, муниципалитеты.

2. **Enterprise-продажи:** Крупные компании (Сургутнефтегаз, Газпром подрядчики) обязаны использовать отечественное ПО. Без реестра — ты за периметром.

3. **Освобождение от НДС:** Экономия 20% на каждой продаже.

4. **Доверие:** В РФ "реестр Минцифры" = знак качества. Это как ISO 9001 для софта.

### ⚠️ Потенциальная проблема: зависимость от зарубежных компонентов

CEOClaw использует:
- Vercel (хостинг) — **иностранный** → нужен вариант на Yandex Cloud
- OpenAI API — **иностранный** → нужны GigaChat/YandexGPT как альтернатива
- Next.js (React) — open-source, **не блокирует**
- PostgreSQL — open-source, **не блокирует**

**Решение:** Для реестра нужно показать, что продукт **может** работать без критических зарубежных зависимостей:
- Деплой на Yandex Cloud (или VK Cloud, Selectel)
- GigaChat/YandexGPT как основные AI-провайдеры (OpenAI — опциональный)
- Это не значит "убрать Vercel" — значит "иметь вариант без Vercel"

## 9. Как не дать себя кинуть

### Правило №1: Никогда не отдавай код

- Демо — на твоём сервере
- Пилот — на твоём сервере
- Партнёрство = реселлинг (продажа доступа), не передача исходного кода
- Даже self-hosted = Docker-образ (скомпилированный), не source code

### Правило №2: Всё фиксируй документально

**Перед каждым демо:**
- NDA (соглашение о неразглашении) — если показываешь что-то уникальное
- Простой NDA на 1 страницу: "Информация о продукте конфиденциальна, не копировать, не воспроизводить"

**Перед каждым пилотом:**
- Договор пилотного внедрения:
  - Срок: 30 дней
  - Стоимость: 0 руб. (или символическая сумма)
  - Обязанности сторон: обратная связь, право на кейс
  - IP остаётся у тебя

**Перед каждой продажей:**
- Лицензионный договор (не договор купли-продажи!)
  - Лицензионный = ты сохраняешь права, клиент получает доступ
  - Договор купли-продажи = ты теряешь права. **Никогда.**

### Правило №3: Проверяй партнёров

**Перед любым партнёрством:**
1. Проверить ОГРН на egrul.nalog.ru
2. Проверить суды на kad.arbitr.ru
3. Проверить на rusprofile.ru — финансы, судебные дела, связи
4. Запросить рекомендации у 2-3 их клиентов
5. Личная встреча > переписка

### Правило №4: Структура оплаты

- **Предоплата 30-50%** за любые услуги внедрения
- **Подписка — помесячная** (не годовая без обеспечения)
- **Escrow** для крупных сделок (через банк)
- Никогда: "мы заплатим после внедрения" (не заплатят)

### Правило №5: Постепенность

```
Знакомство → Демо (30 мин) → Пилот (30 дней, бесплатно) 
→ Платная подписка (помесячно) → Годовой контракт (скидка 20%)
→ Enterprise (индивидуально)
```

Не перескакивай этапы. Не давай годовую скидку без 3+ месяцев оплаты.

```
- vc.ru / Habr — авторы статей по КСУП
- Конференции: PM-Camp, Agile Days, РМ'Expo

```

## 12. Варианты развития — 3 сценария

### Сценарий A: "Bootstrapper" (рекомендуемый)

**Суть:** Сам продаёшь, сам развиваешь. Без инвесторов.

```
Месяц 1-3:  Юридика + доработка + первые 5-10 клиентов
Месяц 3-6:  MRR 60-200K руб, 1-2 партнёра-реселлера
Месяц 6-12: MRR 300K-1M руб, нанять 1 разработчика + 1 продажника
Год 2:      MRR 1-3M руб, команда 3-5 человек, ХМАО + Тюмень + Екатеринбург
```

**Плюсы:** 100% контроль, не зависишь от инвесторов  
**Минусы:** Медленный рост, всё на тебе  
**Когда выбирать:** Если хочешь спокойно расти и сохранить контроль

### Сценарий B: "Акселерация"

**Суть:** ФРИИ/Сколково + грант + первые продажи = быстрый масштаб

```
Месяц 1-2:  Юридика + доработка + подача в ФРИИ Спринт
Месяц 2-5:  Акселератор (12 недель) — менторство, пилоты, демо-день
Месяц 5-8:  Грант 2-4M руб + первые 10-15 клиентов
Месяц 8-12: Нанять команду (3 чел.), MRR 500K-1M
Год 2:      Раунд seed 15-30M руб, масштаб на всю РФ
```

**Плюсы:** Быстрый рост, менторство, связи, деньги  
**Минусы:** Придётся отдать 5-15% доли (ФРИИ/инвесторы)  
**Когда выбирать:** Если хочешь расти быстро и готов делиться

### Сценарий C: "Гибрид" (оптимальный)

**Суть:** Начинаешь как bootstrapper, параллельно подаёшь в ФРИИ. Если возьмут — ускоряешься. Если нет — продолжаешь сам.

```
Неделя 1-4:  Юридика + доработка продукта + первые демо
Месяц 2:    Подать в ФРИИ + "Мой бизнес" + Роспатент
Месяц 2-4:  Первые 3-5 клиентов (своими силами)
Месяц 4-7:  Если ФРИИ — акселерация. Если нет — bootstrapping
Месяц 7-12: В любом случае — MRR 200K-500K, 2-3 партнёра
```

**Это самый безопасный путь.** Ты ничего не теряешь, подавая в ФРИИ параллельно с самостоятельными продажами.

## 13. Честное мнение

### Что меня впечатляет:
- Ты один сделал продукт уровня команды из 5-8 человек
- 167K строк, 54 модели, 179 API, 132 теста — это серьёзно
- Связка AI + GPS + 1C + Telegram + Evidence — реально уникальна для РФ
- Архитектура продуманная (5 позвоночников, connector pattern, workspace isolation)

### Что тревожит:
- **Ты инженер, а не продажник.** Это нормально, но 95% успеха SaaS = продажи. Тебе придётся научиться продавать или найти партнёра, который умеет.
- **Один человек = одна точка отказа.** Если ты заболеешь на месяц — всё остановится. Нужен хотя бы один человек в команде.
- **Длинный цикл B2B в РФ (3-6 месяцев).** Нужна финансовая подушка минимум на 6 месяцев.
- **Консерватизм строителей.** "Зачем нам AI, мы 20 лет в Excel работаем" — будешь слышать часто. Не злись, показывай числа.

### Мой совет, если бы я был на твоём месте:

1. **Следующие 2 недели:** Допили ЮKassa + Landing + Demo. Ничего больше.
2. **Следующие 2 недели:** 20 звонков/сообщений → 10 демо → 2-3 пилота.
3. **Параллельно:** Роспатент, ООО, IT-аккредитация, заявка в ФРИИ.
4. **Через 1 месяц:** Первый платящий клиент. Это психологически перевернёт всё.
5. **Через 3 месяца:** 5-10 клиентов, MRR 50-100K. Понимание, что работает, что нет.
6. **Через 6 месяцев:** Нанять одного человека (продажи или разработка).

**Самая большая ошибка, которую ты можешь сделать — продолжать "допиливать" вместо продажи.**

У тебя уже 93% продукта. Оставшиеся 7% — не инженерные задачи, а бизнесовые (платёжка, лендинг, демо). Не нужен ещё один AI-агент или ещё один дашборд. Нужен первый клиент, который заплатит деньги.
