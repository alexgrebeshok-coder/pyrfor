## 1. Чего не хватает в продукте

*Дополнение к `PROJECT_AUDIT_2026-03-26.md` — фокус на продуктовой ценности*

### 1.1 Критические для продажи (без этого нельзя)

| Отсутствует | Почему блокирует продажи | Приоритет |
|-------------|--------------------------|-----------|
| **Stripe → ЮKassa/CloudPayments** | Stripe в РФ не работает. Нет платёжки — нет денег | 🔴 P0 |
| **Demo-режим без регистрации** | Клиенты в РФ не дают email без живой демонстрации | 🔴 P0 |
| **Онбординг-мастер (wizard)** | Сложный продукт без гида = чёрная дыра для пользователя | 🔴 P0 |
| **Landing page на RU с ценами** | Нет публичного лендинга с ценами → нет входящих лидов | 🔴 P0 |
| **Telegram-бот для поддержки** | Основной канал связи в РФ. Нет бота — нет поддержки | 🟠 P1 |

### 1.2 Необходимые для retention (чтобы клиенты не уходили)

| Отсутствует | Влияние |
|-------------|---------|
| **Email-рассылка (в продукте)** | Нет цикла возвращения пользователей |
| **Уведомления** | Push/email о задачах, дедлайнах, апрувах |
| **Мобильное приложение** | Руководители проектов — часто "в поле" |
| **Шаблоны проектов** | Быстрый старт для новых пользователей |
| **Импорт из Excel/MS Project** | Все клиенты приходят с историей в Excel |
| **Экспорт в Excel/Word** | Отчёты нужны в привычном формате |

### 1.3 Необходимые для продаж enterprise (для крупных клиентов)

| Отсутствует | Влияние |
|-------------|---------|
| **On-premise / self-hosted (Docker)** | Крупные компании РФ не доверяют облаку |
| **SSO через Яндекс 360** | Корпоративная авторизация в России |
| **152-ФЗ соответствие** | Хранение ПД на серверах РФ |
| **Кастомные роли** | Каждая стройкомпания — своя оргструктура |
| **API для интеграций** | Открытый API для 1С-интеграторов |
| **White-label** | Партнёры хотят продавать "свой продукт" |

### 1.4 Важные для отраслевой специфики (стройка + нефтегаз)

| Отсутствует | Зачем нужно |
|-------------|-------------|
| **Сметная интеграция** | Стройка = сметы. Нет — теряем нишу |
| **Строительные шаблоны** | Жилое, коммерческое, дороги, нефтегаз |
| **КС-2, КС-3 шаблоны** | Обязательная документация в стройке |
| **Журнал работ (по ГОСТ)** | Требование контроля строительства |
| **Нормативная база** | ГОСТ Р 54869, ГрК, 44-ФЗ/223-ФЗ |
| **Гео-метки для выездных** | Подтверждение местонахождения прораба |

## 2. Позиционирование

### 2.1 Текущая проблема позиционирования

CEOClaw сейчас — "всё для всех". Это убивает продажи. Нужен чёткий ICP (Ideal Customer Profile).

### 2.2 Предлагаемая стратегия позиционирования: "вертикальные волны"

**Волна 1 (сейчас):** Строительство и инфраструктура  
→ Самый болезненный рынок + уже есть интеграции (GPS, 1C, Telegram)

**Волна 2 (+6 мес):** Нефтегаз и промышленные объекты  
→ ХМАО — идеальный плацдарм (Сургутнефтегаз, Газпром подрядчики)

**Волна 3 (+12 мес):** IT-компании, агентства, проектные офисы  
→ Горизонтальный рост, более дешёвые продажи

**Волна 4 (+18 мес):** Государственные и муниципальные проекты  
→ Реестр отечественного ПО, госзакупки

### 2.3 Ключевое сообщение (elevator pitch)

> **CEOClaw** — это AI-система управления проектами для компаний, где важен контроль в реальном времени:  
> GPS-данные, отчёты с объектов, финансовый факт из 1С — всё в одном месте.  
> Не Битрикс24, где нужно "настроить всё под себя".  
> Работает "из коробки" для строительства и промышленности.

## 4. Техническая доработка под Россию

### 4.1 Платёжная система (P0 — нужна немедленно)

**Выбор: ЮKassa** (YooMoney) — рекомендуется как основная

**Причины:**
- Официальный Node.js SDK: `npm install yookassa`
- Поддерживает: Visa/MC/МИР, СБП, Apple Pay, Google Pay, рекуррентные платежи
- 54-ФЗ фискализация встроена
- Webhook-и для автоматического управления подписками

**Интеграция с Next.js:**
```typescript
// app/api/billing/checkout/route.ts (заменить Stripe на ЮKassa)
import YooKassa from 'yookassa';

const yookassa = new YooKassa({
  shopId: process.env.YOKASSA_SHOP_ID!,
  secretKey: process.env.YOKASSA_SECRET_KEY!,
});

export async function POST(request: NextRequest) {
  const { planId, workspaceId } = await request.json();
  
  const payment = await yookassa.createPayment({
    amount: { value: getPlanPrice(planId), currency: 'RUB' },
    confirmation: {
      type: 'redirect',
      return_url: `${process.env.NEXT_PUBLIC_APP_URL}/billing/success`,
    },
    description: `CEOClaw — план ${planId}`,
    metadata: { workspaceId, planId },
    save_payment_method: true, // для рекуррентных платежей
  });
  
  return NextResponse.json({ confirmationUrl: payment.confirmation.confirmation_url });
}
```

**Также нужно реализовать:**
- Webhook handler: `/api/billing/yokassa-webhook`
- Рекуррентные платежи (auto-renewal)
- Страница управления подпиской
- Страницы success/cancel/error

**Резервный вариант: CloudPayments** (если ЮKassa откажет)
- Web SDK для клиентской стороны
- REST API для серверной части

## 6. Go-to-Market стратегия

### 6.1 Целевые сегменты (ICP)

**Сегмент A — Приоритетный:**
- Строительные подрядчики (50–500 сотрудников)

- Нефтегазовые подрядчики (ХМАО, ЯНАО)

**Сегмент B — Вторичный:**
- IT-компании с проектным производством
- Промышленные предприятия

**Сегмент C — Будущее:**
- Управляющие компании ЖКХ
- Энергетические компании
- Аграрный сектор

**Почему:** У каждого 1С-франчайзи есть база строительных клиентов. Они уже "в дверях" у нашего ICP.

**Как найти в Сургуте:**
- Поиск "1С:Франчайзи Сургут" → ~5–10 компаний
- Прямые переговоры с руководством

**Формат участия:**
- Выступление на отраслевых мероприятиях
- Статус "технологического партнёра"

## 10. Юридические и технические требования для РФ

### 10.1 Юридическая структура

| Вопрос | Рекомендация |
|--------|--------------|
| **ОПФ** | ООО или ИП. ООО лучше для B2B (корпоративные контракты) |
| **Налоги** | УСН 6% (с дохода) или 15% (доходы-расходы). Для ИТ — льготы |
| **ИТ-льготы** | Пониженные страховые взносы (7.6% вместо 30%) при аккредитации в Минцифры |
| **НДС** | ПО исключено из НДС при правильном оформлении лицензионного договора |
| **Оферта** | Публичная оферта на сайте + Политика конфиденциальности (обязательно) |

### 10.2 152-ФЗ (Персональные данные)

**Обязательно:**
- Политика конфиденциальности на сайте
- Согласие на обработку ПД при регистрации
- Хранение ПД граждан РФ на серверах в РФ (при наличии enterprise-клиентов)
- Уведомление Роскомнадзора (если обрабатываете ПД)

**Хостинг для compliance:**
- Yandex Cloud (дата-центры в России)
- VK Cloud (аналог)
- Selectel (петербургский хостинг)

### 10.3 54-ФЗ (Онлайн-касса)

При приёме платежей обязательна онлайн-касса.  
**Решение:** ЮKassa имеет встроенную фискализацию — не нужно отдельно покупать кассу.

### 10.4 Техническая инфраструктура для РФ

| Компонент | Текущее (Vercel) | Для РФ enterprise |
|-----------|------------------|-------------------|
| Hosting | Vercel (Frankfurt) | Yandex Cloud / VK Cloud |
| Database | Neon PostgreSQL (US/EU) | Managed PostgreSQL в РФ |
| CDN | Vercel Edge | Яндекс CDN / CloudFlare |
| Email | Nodemailer | + Unisender / SendPulse |
| Object Storage | — | Yandex Object Storage |
