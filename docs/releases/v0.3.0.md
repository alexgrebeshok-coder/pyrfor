# Pyrfor v0.3.0 — Universal Engine Release Candidate

Pyrfor v0.3.0 closes the Universal Engine release-candidate line and makes the governed OpenClaw migration path operational.

## Highlights

- Universal Engine governance stack is complete through release-critical M1-M18 / R1-R7 scope.
- R0 release trust is closed with the signed `ue-governance-baseline-m1` GPG baseline tag.
- Memory v2 import safety is active: imported memory is quarantined, provenance-tagged, contradiction-checked, and planner-hidden until approved.
- Postmortem and historian wiring now persist lessons through the governed memory path.
- Desktop Operator Console MVP includes trust, orchestration, migration, approvals, concept trace, timeline, and postmortem/lessons surfaces.
- VSCode approval inbox and concept trace panels are wired for operator review.
- OpenClaw migration can proceed as a governed/operator-supervised RC operation.

## Release hardening

- Fixed file-backed SQLite memory-store startup on fresh machines by creating parent directories before opening the database.
- Added regression coverage for nested file-backed memory-store paths.
- Stabilized runtime approval/session persistence tests.
- Regenerated the Tauri updater key after discovering the old private key was missing locally and absent from GitHub Actions secrets.
- Updated updater URLs and `latest.json` generation to the actual release repository: `alexgrebeshok-coder/pyrfor`.
- GitHub Actions releases now require the Tauri updater signing key and can publish updater-signed RC artifacts even when Apple Developer notarization secrets are not configured.

Observed final gate result: full tests/typecheck/build/readiness/platform-release checks passed.

## Known release posture

This is a release-candidate/beta-grade desktop release. Apple Developer ID signing/notarization is not configured in GitHub Actions yet, so macOS may show Gatekeeper warnings for downloaded app artifacts. The updater bundle itself is signed with the Tauri updater key stored in GitHub Actions secrets.

R8 full MCP/A2A product routing and R9 full cloud deployment remain roadmap expansion tracks and do not block the v0.3.0 release or governed OpenClaw migration.
