# v0.4.0-rc.1 (release candidate)

> **Do not tag this release without explicit approval.** This document lists the human steps to ship after trust-layer and Block SDK PRs merge.

## Highlights

- Permission ladder enforced on `executeRuntimeTool`
- Sandbox L1 bootstrap from `runtime.json`
- Self-improvement run budget guard before `MetaCritic.run`
- Block executor + KS reconciliation gateway APIs
- Fixture gate: `pnpm validate:block-fixtures`

## Human checklist

1. Merge PRs in order: #35 → #36 → #37 → #38 → #39 → #40 (or squash stack to `main`).
2. Merge [PR #34](https://github.com/alexgrebeshok-coder/pyrfor/pull/34) when CI checks appear and pass (signing/notarization).
3. Configure Apple signing secrets for notarized macOS DMG (see `docs/RELEASE.md`).
4. Point DNS **docs.pyrfor.dev** CNAME to GitHub Pages or Vercel (`docs-site/DEPLOY.md`).
5. Run locally:
   - `pnpm test`
   - `pnpm qa:first-run`
   - `pnpm release:check`
   - `pnpm validate:block-fixtures`
6. Tag only after the above are green:
   ```bash
   git tag v0.4.0-rc.1
   git push origin v0.4.0-rc.1
   ```
7. Publish GitHub Release from `pyrfor-release.yml` workflow artifacts.
