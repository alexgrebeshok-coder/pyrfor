# Governance

Pyrfor uses a **permission ladder** and **budget policies** on governed runs:

- Tools declare capabilities; the permission engine blocks undeclared effects.
- Budget guardrails abort runs that exceed token/cost limits.
- Approvals and the event ledger provide an audit trail.

Contributing: see `CONTRIBUTING.md` and `CODE_OF_CONDUCT.md` in the repository root.

Security: report issues via GitHub Security Advisories (see `SECURITY.md`).
